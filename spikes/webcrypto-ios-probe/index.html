Tyde PSK storage probe

PHASE-0 SPIKE. Open this on the target iOS Safari (and again after "Add to Home Screen") to validate the hardened PSK-storage decision before the async-HKDF refactor. It:

  1. imports a non-extractable HKDF CryptoKey from 32 raw bytes,
  2. derives 256 bits via deriveBits (HKDF-SHA256) using Tyde's real salt/info strings, and checks the result is byte-identical across reloads,
  3. persists the CryptoKey itself in IndexedDB (structured clone),
  4. reloads it from IndexedDB and re-derives to prove the stored key still works,
  5. verifies the stored key is still non-extractable (exportKey must throw),
  6. reports navigator.storage.persist() / persisted() support.

The acceptance test for iOS: run "Step 1" once, force-quit, reopen, run "Step 2" — the derived bits must match and the key must reload. Repeat from the Home-Screen-installed PWA, and again after >7 days to check ITP eviction.
