# Getting Started

Tyde is a desktop workspace for coding agents. It provides a chat interface, visual diffs, git staging and committing, a file explorer, built-in terminals, and multi-agent management — connected to the coding agent of your choice. You can run multiple agents at the same time within a single project, or work across multiple projects simultaneously, each with its own independent set of agents, tabs, terminals, and layout. Tyde also extends coding agents with capabilities they don't provide on their own, like multi-agent orchestration and worktree management.

## Backends

A backend is the coding agent that Tyde connects to. Tyde currently supports four: **Tycode** (Tyde's native backend), **Claude Code** (Anthropic), **Codex** (OpenAI), and **Kiro** (Amazon). Each backend has its own models, tool implementations, and capabilities. Tyde communicates with all of them through a common protocol, so the workspace features work the same regardless of which one you choose. You can also mix backends within a single workspace — run a Claude Code agent alongside a Codex agent on the same project, compare their results, or have agents from different providers collaborate on the same codebase.

To set your default backend, go to **Settings → Backend** and select from the dropdown. You can also run different backends on different workspaces.
